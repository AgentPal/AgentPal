# Contributing to AgentPal

Thank you for helping improve AgentPal.

## Ways to contribute

- Improve public documentation.
- Improve Pal Pack protocols.
- Add release-safe examples.
- Propose new templates.
- Improve Mira's public guidance files.
- Add compatibility notes for agent runtimes.

## Contributing Pals

Pal contributions should be valid Pal Packs with:

- `PAL.md`
- `SKILL.md`
- `AGENTS.md`
- `pal.json`
- clear identity and responsibility boundaries
- collaboration permissions
- safety and verification notes

Use `Name-role` directory names, such as `Atlas-developer`.

## Do not submit private data

Do not submit:

- private user memory
- private project information
- internal development notes
- API keys, tokens, passwords, secrets, or credentials
- copied internal reference documents
- long third-party README, Skill, or source text

## Contacts boundary

Do not add ordinary Skills, tools, models, MCP servers, plugins, non-Pal runtimes, or raw repositories to Pal contacts. Only valid Pal Packs can enter contacts.

## Documentation style

Prefer clear user-facing English. Chinese notes are welcome where they help users understand important rules.

AgentPal v0.1 must remain usable without Python, Node.js, Rust, scripts, installers, CLIs, services, or UI code.

## Release boundary

Everything under the public AgentPal repository is treated as release content. Keep internal development reports, private user memory, private project state, and secrets outside the repository.

Third-party resources keep their original licenses. Do not copy third-party source code or long documentation text into AgentPal.
