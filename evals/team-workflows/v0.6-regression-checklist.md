# v0.6 Team Workflow Regression Checklist

Status: lightweight manual regression checklist.

This checklist is a prevention anchor for AgentPal v0.6 Team Pack, Workflow
Execution Contract, Contact Capability Card, Routing Veto, Pal naming, and
Asset Preflight work.

It is not an automated test framework, scanner, validator, CLI, background
service, runtime scheduler, database, or UI. Passing this checklist does not
prove real host-runtime execution. It only helps maintainers avoid common
documentation, template, and example regressions.

## When To Use

Use this checklist before accepting changes that touch:

- Team Pack standards, templates, examples, reports, or rosters;
- Workflow Execution Contract, Step state machine, or Closure Gate files;
- Pal Asset Preflight or Team Asset Preflight files;
- contacts, capability cards, routing veto, or roster references;
- PalSmith creation templates, naming rules, or Team Pack creation examples;
- Mira routing / team coordination entry files;
- docs or release notes that describe current runtime behavior.

## Example Failure Cases

### Failure 1: Missing Roster Pal

Wrong:

```json
{
  "pal_id": "luma-visual",
  "display_name": "Luma",
  "team_roles": ["visual-designer"]
}
```

Why it fails: Luma is not currently registered in central contacts and no
official Luma Pal Pack exists. Use an `open_roles` entry unless a complete Pal
creation and contact registration task is approved.

### Failure 2: Role-Title-Only Pal Name

Wrong:

```json
{
  "display_name": "文案 Pal",
  "role_title": "copywriter"
}
```

Why it fails: `display_name` must be a human name. Use a name such as `Harper`
or a user-approved new human name, and store `文案顾问` or `copywriter` as
`role_title`.

### Failure 3: Faye Reassigned To Existing Team Daily Execution

Wrong:

```text
The after-sales team already exists, so Faye will execute today's customer
feedback replies.
```

Why it fails: Faye can design or redesign the business workflow, but routine
daily execution after a team exists should go to the Team Pack owner and member
Pals. Faye may consult only if the workflow itself is being reworked.

### Failure 4: Vanishing Quinn Verifier

Wrong:

```text
Workflow Execution Contract:
- step-04 verifier: Quinn

Final report:
All work is complete.
```

Why it fails: If Quinn is written into the contract, the final report must show
Quinn's verifier output, or mark the verifier Step as skipped, blocked, not-run,
cancelled, failed, or replanned with a reason.

### Failure 5: Copying Team Pack Into External Project

Wrong:

```text
Install AgentPal by copying the full Team Pack and all Pal assets into the
external project's .agentpal directory.
```

Why it fails: external projects should stay thin-bound. Team Packs and Pal Packs
remain in the AgentPal Workspace unless a separate export / installation task is
explicitly approved.

### Failure 6: Runtime Overclaim

Wrong:

```text
AgentPal v0.6 automatically runs team workflows in parallel and closes all
steps.
```

Why it fails: v0.6 assets are Markdown / JSON protocols, templates, and
rehearsal examples. They do not implement runtime automation.
