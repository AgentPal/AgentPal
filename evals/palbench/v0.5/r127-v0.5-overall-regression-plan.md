# R127 - v0.5 Overall Regression Plan

## Status

Planning status: complete.

Execution status: not-run. R127 defines the v0.5 overall regression plan only. R128 is the recommended execution round.

Decision target: prepare a bounded, evidence-based regression pass for the current v0.5 local development line before any v0.5 local completion report, tag, push, or GitHub Release work.

## Purpose

This plan turns the v0.5 mainline into an executable regression package. It connects the v0.4 baseline, Pal Asset and PalSmith standards, the PalSmith metadata/manifest pilot, Org Pack, FDE Pack, Asset Boundary, Combined Pack, Usage Scenario, Workflow/Governance examples, and public safety checks into one regression sequence.

The plan is designed to catch:

- invalid JSON or broken repository navigation;
- thin-binding drift that writes shared assets into external project `.agentpal/` directories by default;
- central roster or fixed keyword routing regressions;
- official Pal metadata drift beyond the current PalSmith pilot;
- Org/FDE/Combined Pack example inconsistencies;
- customer-private, credential, connector, scanner, marketplace, or runtime-program expansion leakage;
- release-readiness overclaims.

## Scope

The R128 regression execution should cover these 12 groups:

| Group | Name | Main evidence focus | Severity floor |
| --- | --- | --- | --- |
| 1 | Repository / JSON / clean-copy | JSON parse, required paths, clean-copy usability | blocker |
| 2 | External project thin binding | external project `.agentpal/` remains a thin binding only | blocker |
| 3 | Central roster / no keyword routing | 9 central contacts, no hard-coded semantic route maps | blocker |
| 4 | Official Pals / Pal Asset state | 9 official Pal directories, no unplanned official metadata rollout | high |
| 5 | PalSmith metadata / manifest pilot | PalSmith `pal.json` and manifest pilot remain bounded | high |
| 6 | Pal Asset / PalSmith standards | standards/templates/examples remain no-code and public-safe | high |
| 7 | Org Pack standards / templates / examples | Org Pack structure and examples remain reusable assets | high |
| 8 | FDE Pack standards / templates / examples | FDE Pack remains expert-delivery guidance, not a connector/runtime | high |
| 9 | Asset Boundary | reusable vs customer-private boundary remains explicit | blocker |
| 10 | Combined Pack | shared-entry composition remains reference-only and review-gated | high |
| 11 | Usage Scenario / Workflow / Governance | walkthroughs and governance examples remain no-code examples | medium |
| 12 | Public safety / no program expansion | no credentials, customer-private data, scanner, connector, marketplace, tag, push, or release | blocker |

## Out Of Scope

- Running the full v0.5 regression in R127.
- Fixing substantive regressions found by R128 unless they are tiny, low-risk documentation or JSON pointer fixes that R128 explicitly classifies as safe.
- Creating runtime code, scanners, validators, installers, daemons, connectors, marketplace flows, API clients, automatic route engines, or external Agent calls.
- Updating central roster membership or official Pal `pal.json` files.
- Creating tags, pushing to GitHub, publishing releases, or claiming v0.5 completion.
- Moving customer/private project material into public release files.

## Execution Order For R128

1. Preflight: confirm no push, pull, fetch, tag, or release is needed for the regression execution.
2. Parse all visible JSON files that belong to the checked surface.
3. Verify required R127 plan artifacts and v0.4 baseline evidence paths exist.
4. Create a clean local copy that excludes `.git`, generated caches, and dependency folders, then repeat key path and JSON checks inside the copy.
5. Simulate thin binding by checking templates and examples for central-workspace references and external-project write boundaries.
6. Check central roster count and no-keyword-routing policy.
7. Check official Pal directories and Pal Asset metadata state, including the bounded PalSmith pilot.
8. Check Pal Asset and PalSmith standards, templates, and examples.
9. Check Org Pack, FDE Pack, Asset Boundary, Combined Pack, Usage Scenario, Workflow, and Governance examples.
10. Run public safety scans on the R128 evidence surface and any changed files.
11. Classify every finding with the R127 issue template.
12. Produce R128 results, issues, validation, and readiness decision.

## Required Evidence

R128 should produce or update these evidence classes:

- command transcript or summarized command output for JSON parse, file existence, directory counts, and clean-copy checks;
- explicit path list for every required artifact;
- central roster count and official Pal directory count;
- protected diff for central roster and official Pal metadata paths;
- keyword-routing scan results;
- scanner/connector/marketplace/runtime-program expansion scan results;
- credential and customer-private leakage scan results;
- issue table using `release/integration-notes/r127-v0.5-overall-regression-issue-template.md`;
- final R128 readiness decision.

## Pass Criteria

The v0.5 overall regression passes only if:

- all 12 groups have an executed result;
- all required JSON files parse;
- the clean-copy check passes;
- central roster remains 9/9 and official Pal directories remain 9;
- no active fixed keyword-routing map is introduced;
- no unplanned official Pal metadata or manifest rollout occurs beyond the PalSmith pilot;
- no customer-private or credential material is present in public reusable assets;
- no scanner, connector, marketplace, daemon, installer, API client, or runtime program expansion is introduced;
- all blocker/high findings are either absent or moved to a dedicated fix round before completion;
- any medium/low findings are documented with an explicit fix or deferral decision.

## Failure Severity

| Severity | Meaning | R128 handling |
| --- | --- | --- |
| blocker | Invalid release boundary, broken JSON, missing core evidence, customer-private/credential leak, unsafe external write, tag/push/release attempt, or runtime-program expansion | Stop completion claim. Recommend R129 fix round. |
| high | Official Pal/PalSmith/Org/FDE/Combined Pack contract drift that could mislead users or downstream runtimes | Usually recommend R129 fix round unless the fix is tiny and isolated. |
| medium | Documentation, navigation, or evidence mismatch that does not break safety boundaries | May be fixed in R128 if low-risk and scoped. |
| low | Typo, wording, or non-blocking index mismatch | May be fixed in R128. |
| note | Observation without required change | Record only. |

## Fix Policy

R128 may fix only small, evidence-backed text or JSON pointer issues when all of the following are true:

- the finding is medium or low;
- the edit does not change standards, runtime behavior, roster membership, official Pal metadata, or release status;
- the edit is required to make the regression evidence internally consistent;
- the R128 results file records the fix.

Blocker or high findings should normally move to R129 as a dedicated fix round.
