# AgentPal v0.2 Public Capability Summary

This is a restrained public-facing summary of what the v0.2 first phase adds.

## What v0.2 Strengthens

AgentPal v0.2 strengthens the no-code Pal layer in six areas:

- PalSmith end-to-end creation flows for a first professional Pal and a small AI team.
- Mira-first usage for users who want to start with one default Main Pal.
- Official Pal task examples for all 9 bundled Pals.
- Minimal Capability Inventory profiles as manual AI judgement inputs.
- PalBench Light as a qualitative release regression suite.
- Runtime Adapter troubleshooting and thin binding guidance.

## What Users Can Do

Users can:

- ask PalSmith to prepare a copyable task package for drafting a Pal or AI team;
- ask Mira to organize ordinary tasks, composite deliverables, and completion reviews;
- inspect realistic examples for Mira, Atlas, Vega, Rhea, PalSmith, Quinn, Morgan, Harper, and Nova;
- use Capability Inventory examples to inform judgement while still requiring current runtime evidence;
- use PalBench Light cases to review AgentPal-mode behavior before release;
- use thin binding guides and prompt cards when connecting AgentPal to Codex, Claude Code, or a generic CLI agent.

## What v0.2 Does Not Claim

v0.2 does not claim:

- automatic Deep Conductor execution;
- autonomous multi-agent runtime behavior;
- automatic detection of every runtime, model, plugin, MCP tool, or Skill;
- statistically significant performance proof;
- replacement of Codex, Claude Code, OpenHands, or any host runtime;
- PalSmith as a CLI, scanner, importer, exporter, builder, or installer.

## Current Boundary

AgentPal remains a Markdown / JSON / protocol workspace. Pals organize identity, judgement, context, Task Packages, examples, and verification. The host runtime performs real file reads, writes, commands, tool calls, rendering, publishing, and other actions, and must provide evidence.

## Readiness Status

v0.2 first-phase integrated readiness: pass / candidate for `v0.2.0-rc.1` release preparation.

This does not mean production complete. It means the first integrated documentation, examples, evals, and Runtime Adapter guidance are coherent enough to begin release-candidate preparation.
