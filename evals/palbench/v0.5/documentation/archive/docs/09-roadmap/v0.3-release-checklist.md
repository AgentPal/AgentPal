# v0.3.0-rc.1 Release Preparation Checklist

Use this checklist before starting `v0.3.0-rc.1` tag, push, or GitHub Release work.

This checklist does not authorize publishing by itself.

## Readiness Documents

- [ ] `evals/palbench/v0.5/documentation/archive/docs/09-roadmap/v0.3-deep-conductor-readiness.md` recommends entering `v0.3.0-rc.1` release preparation.
- [ ] `evals/v0.3-integration/v0.3-deep-conductor-integration-test-matrix.md` records the integrated status matrix.
- [ ] `evals/palbench/v0.5/documentation/archive/docs/09-roadmap/v0.3-public-capability-summary.md` states current claims and forbidden claims.
- [ ] `evals/palbench/v0.5/documentation/archive/docs/research/palbench-collaboration-coverage-audit.md` records coverage and remaining gaps.

## Public Boundary

- [ ] v0.3 is described as a no-code orchestration prototype line.
- [ ] Deep Conductor is described as no-code coordination methodology, not an executor.
- [ ] Runtime Skill Orchestration is described as host Runtime candidate handling, not AgentPal Skill execution.
- [ ] Context Budget is described as qualitative, not exact token or cost metering.
- [ ] Subagent / external Agent support is described as unavailable / unknown / host-dependent unless current Runtime evidence exists.
- [ ] PalBench Collaboration is described as qualitative regression coverage, not statistical benchmark evidence.

## Required Assets

- [ ] Deep Conductor master goal, master loop guide, protocol, Project Conductor workflow, E2E guide, E2E package, and synthesis report exist.
- [ ] Context Packet, Context Access List, Owner + Verifier, Parallel Independent Review, and Plan -> Execute -> Verify assets exist.
- [ ] Runtime Skill candidate, availability, fallback, and usage-memory assets exist.
- [ ] Cross-Runtime Pal Memory, memory boundary, routing memory, and continuation assets exist.
- [ ] Context Budget, Context Usage Report, and prompt shaping assets exist.
- [ ] Subagent / external Agent boundary, external handoff package, and subagent task package exist.
- [ ] PalBench Collaboration task index includes all R51-R62 cases.

## Verification

- [ ] JSON parse check passes for public JSON files.
- [ ] Public changed files are Markdown / JSON documentation assets only.
- [ ] No runtime code, scripts, CLI, scanner, validator, installer, daemon, service, database, desktop app, web app, token meter, or cost calculator was added.
- [ ] No fixed Pal, Runtime, Skill, model, topology, keyword, domain, or file-extension routing rule was added.
- [ ] No future capability is described as current execution behavior.
- [ ] No internal local path, private report, raw private memory, API key, token, password, credential, or secret is present in public release files.
- [ ] `git diff --check` passes before commit.
- [ ] `git status --short` is clean before tag / push / release work.

## Publication Hold

- [ ] Tag creation is not done until explicitly authorized.
- [ ] Push is not done until explicitly authorized.
- [ ] GitHub Release is not created until explicitly authorized.
- [ ] Release notes preserve all `partial`, `unavailable`, `not-run`, and host-dependent limitations.
