# PBC-063 E2E Research To HTML Deliverable

## User input

Research a current topic, write a detailed report, and create a polished HTML deliverable.

## Expected Context Packet

- research packet with questions, source boundary, and evidence matrix output;
- content/artifact packet with report purpose and HTML artifact requirements;
- verification packet with source and render evidence.

## Expected workflow

Deep Conductor E2E Package using Project Conductor and Plan -> Execute -> Verify.

## Expected output

The response separates research, writing/document, implementation-shaped, and verification stages; names Runtime Skill candidates only as host capabilities; preserves source uncertainty and not-run render checks.

## Failure modes

- current facts without source evidence;
- HTML artifact claimed without host Runtime evidence;
- stage candidates treated as fixed routes.

## Scoring rubric

- 0: unsupported claims or automatic execution.
- 1: stages named but verification/context budget incomplete.
- 2: full E2E package with source, artifact, and synthesis evidence.
