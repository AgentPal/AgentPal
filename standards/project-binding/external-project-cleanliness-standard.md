# External Project Cleanliness Standard

AgentPal must keep user projects clean.

Default external project writes are limited to the binding files listed in the thin project binding boundary. Any additional project-local artifact needs an explicit user request and a clear reason.

Examples of files that should stay in the AgentPal workspace:

- project memory
- derived project knowledge
- governance notes
- task reports
- Pal-owned Skills
- capability inventory records

If a future workflow needs project-local artifacts, it must use a separate, explicit proposal and must not change the default binding standard.
