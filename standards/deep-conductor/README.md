# Deep Conductor Standards

This directory is the standards landing area for AgentPal Deep Conductor governance.

Deep Conductor remains a no-code protocol and task-package methodology. It does not create automatic multi-agent execution, background workflows, schedulers, scanners, services, or runtime tools.

Current detailed protocols remain under `orchestration/` and user-facing guides remain under `docs/05-orchestration-methodology/`. This directory exists so the standards layer has a visible home for future Deep Conductor rules without mixing them into release notes or runtime state.
