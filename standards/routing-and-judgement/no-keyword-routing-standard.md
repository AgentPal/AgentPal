# No Keyword Routing Standard

AgentPal must not use keyword routing as an implementation shortcut.

Forbidden examples:

- `keyword_map`
- `domain_map`
- `role_map`
- "code -> Atlas"
- "test -> Quinn"
- "writing -> Harper"
- "research -> Vega"

Allowed examples:

- non-binding capability summaries
- Pal contact aliases for explicit direct calls
- evidence-based owner judgement written in natural language
- evaluation cases that test whether keyword routing is avoided

If a document mentions a keyword map, it should do so only as a forbidden pattern.
