# Research

## Purpose

This directory lists source gathering, source inventories, evidence extracts,
coverage matrices, investigations, comparison notes, and unverified findings
owned by Mira.

Mira research preserves source context for team-leadership, context packaging,
handoff, progress reporting, memory stewardship, and result explanation before
stable conclusions are promoted into knowledge, skills, workflows, runbooks, or
evals.

## What Belongs Here

- Source inventories.
- Source coverage matrices.
- Evidence notes and investigation notes.
- Comparative research drafts.
- Public-safe source summaries.
- Research gap notes.
- Links to related knowledge, skills, workflows, runbooks, examples, evals, and
  reports.

## What Does Not Belong Here

- Final stable rules without source context.
- Long-term memory or user preference summaries.
- Runtime state or current mutable task state.
- Full completion reports or audit reports.
- Connector implementations, runtime clients, scanners, validators, daemons,
  installers, databases, or external execution tools.
- Credentials, tokens, cookies, passwords, API keys, secrets, customer data, or
  private project evidence.
- External project `.agentpal/research/` copies by default.
- Keyword routing maps, domain maps, role maps, or deterministic dispatch
  tables.

## Promotion Path

Research conclusions become stable assets only after review:

- stable concepts -> `knowledge/`
- repeatable Mira-owned methods -> `skills/`, `workflows/`, or `runbooks/`
- acceptance checks -> `evals/`
- extracted long-term continuity lessons -> `memory/`
- task completion evidence -> `reports/`

Preserve `unknown`, `not-run`, and `needs-review` for claims that have not been
verified.

## Relationship To Knowledge

Research is not knowledge until source context is reviewed, private details are
removed, and stable conclusions are summarized. Knowledge should cite or point
back to research when source context matters.

## Relationship To Memory

Research is not memory. Memory stores extracted long-term lessons and approved
coordination preferences. Research may produce memory candidates only after a
stable lesson is identified and reviewed.

## Public Safety Boundary

Do not include private sources, credentials, tokens, cookies, passwords, API
keys, secrets, customer identifiers, copyrighted source excerpts beyond allowed
use, or private project evidence in public research indexes.

## External Project Boundary

Do not copy Mira research into an external project `.agentpal/` directory by
default. External projects remain thin-bound; reusable Mira research belongs in
the central Mira Pal Pack or approved central records.
