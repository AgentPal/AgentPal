# AI Design Tool Discovery

Date checked: 2026-06-30

Purpose: initial research seed for Luma creation. This is not a permanent route map.

## Findings

- Product Design in Codex is best treated as a host plugin with focused workflows: design brief, ideation, audit, image-to-code, prototype, design QA, and sharing. Local plugin docs require the design brief gate before ideation/build and a selected visual target before image-to-code.
- Figma Dev Mode MCP Server is an official Figma integration candidate for providing design context to coding agents when Figma access exists. Source: https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Dev-Mode-MCP-Server
- Framer AI is a web/landing-page generation candidate, useful for rapid marketing-page exploration when current account/tool access is available. Source: https://www.framer.com/ai/
- Recraft is a design/image generation candidate that may help with brand imagery, vector-like illustration, and generated assets. Source: https://www.recraft.ai/
- Penpot is an open-source design/prototyping platform candidate for open design workflows. Source: https://penpot.app/
- Open-source screenshot-to-code, Figma MCP community servers, tldraw/make-real, Remotion, HyperFrames, and GSAP are useful categories to refresh per task rather than permanent fixed dependencies.

## Luma Policy

- Record tool candidates with source, license, maturity, privacy risk, and availability evidence.
- Do not claim a tool was invoked unless there is current runtime evidence.
- Do not hard-code tool lists as permanent facts.
- Prefer Product Design for UI/product workflows, ImageGen for visual assets, Figma for design-system context, and HTML/CSS/Atlas for implementable prototypes when each is currently available.
