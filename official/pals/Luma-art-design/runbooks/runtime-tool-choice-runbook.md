# Runtime Tool Choice Runbook

## Use When

The user asks what tool, plugin, Skill, MCP, model, or external service should be used for a visual design task.

## Steps

1. Identify the deliverable.
2. Check whether the task requires current tool availability evidence.
3. Choose likely candidates:
   - Product Design for UI/product workflows, visual ideation, audits, and prototypes.
   - ImageGen for image assets, concepts, covers, logos, and style frames.
   - Figma tools for canvas/design-system context when access exists.
   - HTML/CSS for responsive visual prototypes.
   - External tools for specialized generation only after current review.
4. State evidence required before claiming execution.
5. State fallback if the tool is unavailable.

## Stop Conditions

- Tool access requires credentials or payment.
- The user asks for publication or external writes without authorization.
- The design needs legal/trademark judgement.
