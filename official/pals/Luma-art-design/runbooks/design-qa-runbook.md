# Design QA Runbook

## Steps

1. Confirm the intended surface and audience.
2. Inspect the latest visual evidence.
3. Check hierarchy, spacing, type, color, contrast, imagery, brand, mobile fit, and over-decoration.
4. Identify P0/P1/P2/P3 issues.
5. Name what was not checked.
6. Recommend fixes.
7. If a runtime build happened, require screenshot or render evidence.

## Acceptance

The review is acceptable when every critical issue has a concrete fix and every unavailable check is marked `not-run` or `blocked`.
