# Design Tool, Skill, Plugin, MCP, And Open-Source Candidate Index

Status: dated research seed
Last checked: 2026-06-30
Refresh rule: update before relying on availability, pricing, API support, platform specs, or integration claims.

This file is a candidate index, not a permanent fact table and not a route map.

## Product And UI Design

- Codex Product Design plugin: local Codex plugin for design brief gathering, product-flow audit, visual ideation, prototype/image-to-code workflows, design QA, and sharing. Availability must be checked in the current host runtime.
- Figma Dev Mode MCP Server: official Figma MCP bridge for Dev Mode context; useful when a task needs Figma design context or code handoff and current Figma access exists. Source: https://help.figma.com/hc/en-us/articles/32132100833559-Guide-to-the-Dev-Mode-MCP-Server
- Framer AI: AI-assisted website generation and design iteration candidate for marketing pages. Source: https://www.framer.com/ai/
- Penpot: open-source design and prototyping platform candidate for self-hosted or open design workflows. Source: https://penpot.app/

## Image, Logo, And Brand Generation

- ImageGen: built-in image generation candidate for visual concepts, moodboards, hero images, covers, style frames, and image assets.
- Recraft: AI design/image tool candidate for vector-like brand assets, illustrations, and generated design imagery. Source: https://www.recraft.ai/
- Adobe Firefly, Canva, Runway, Midjourney, Ideogram, Krea, Leonardo, and similar tools may be useful candidates, but Luma must refresh current capabilities before relying on them.

## Open-Source / Developer-Adjacent Candidates

- screenshot-to-code style projects: useful as reference for visual-to-frontend workflows when current repository quality and license are checked.
- Figma MCP community servers: useful only after current repo, license, and security review.
- tldraw / make-real style workflows: useful for sketch-to-prototype exploration when current availability is verified.
- Remotion / HyperFrames / GSAP: useful for visual systems and motion/video style packs when a video or animated asset is required.

## Workflow Rule

For any current task, Luma should record:

- tool candidate;
- why it fits;
- capability source;
- current availability evidence;
- privacy or external write risk;
- cost/API/key requirement;
- fallback if unavailable;
- whether execution was `real_call`, `dry_run`, `prompt_inspection`, `not_invoked`, or `blocked`.
