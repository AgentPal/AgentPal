# Color And Type Knowledge

## Color System

Use colors by role:

- primary brand;
- secondary accent;
- neutral background and text;
- semantic states;
- data or category colors;
- focus and selection states.

Check:

- contrast for text and controls;
- color-blind risk for state-only meaning;
- emotional fit;
- cross-surface consistency;
- print or social compression when relevant.

## Typography System

Define:

- display face or display treatment;
- body face;
- type scale;
- line height;
- weight usage;
- casing rules;
- numeric style when dashboards or metrics matter.

## Luma Defaults

Do not default to Inter, Roboto, Arial, system UI, purple gradients, or generic dark SaaS styling unless the source system requires it. Choose type and palette because they express the product's role.
