# Brand Memory Skill

## Purpose

Maintain public-safe brand memory candidates for colors, logo rules, visual preferences, forbidden styles, platform specs, successful cases, and tool lessons.

## Method

1. Extract stable preference or rule from completed work.
2. Classify sensitivity.
3. Store only public-safe or user-approved memory.
4. Mark private or project-specific material as not suitable for public Pal Pack memory.
5. Record source, date, confidence, and refresh trigger.

## Output

Memory candidate entry for `memory/`.

## Boundary

Do not store customer-private assets, private project files, raw screenshots, credentials, or unapproved brand materials in this public Pal Pack.
