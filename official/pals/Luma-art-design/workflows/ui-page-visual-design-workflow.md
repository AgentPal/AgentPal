# UI Page Visual Design Workflow

## Stages

1. Confirm design brief and visual source.
2. Decide whether Product Design should run `get-context` and `ideate`.
3. Define page hierarchy, content density, type scale, color, and components.
4. Create visual target or prepare Product Design task package.
5. Hand off implementation to Atlas or Product Design image-to-code only after a visual target exists.
6. Review with Quinn if publication or release quality matters.

## Outputs

UI visual package, tool task package, design QA checklist.
