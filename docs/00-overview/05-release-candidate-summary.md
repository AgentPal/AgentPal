# Release Candidate Summary

AgentPal v0.5 is the current release-candidate line for the AgentPal Workspace, Pal Pack practice, and no-code Pal collaboration in existing host runtimes.

## The Short Summary

- AgentPal is a lightweight Pal team organization layer.
- The workspace is Markdown / JSON / protocol based.
- Codex is the verified first host path.
- Mira is the default Main Pal, Leader, and Conductor.
- The simple Mira-to-owner path remains the normal route for clear work.
- Deep Conductor is available as a no-code collaboration and mode-decision protocol, not automatic runtime execution.
- The current official bundled Pals are Mira, Atlas, Nova, Faye, Vega, Quinn, Morgan, Harper, Rhea, and PalSmith.

## Ready For Current Use

- Open the AgentPal Workspace in Codex and initialize with `prompts/codex/initialize-agentpal-workspace.md`.
- Use thin binding for external projects.
- Use Claude Code only under the documented limited/minimal handoff claim.
- Use Generic CLI Agent only as a compatibility path.

## Important Boundaries

- AgentPal is not an Agent runtime, multi-Agent runtime, app runtime, or execution layer.
- AgentPal does not include a CLI installer, scanner, daemon, connector layer, marketplace, or hosted service.
- Subagent launch, parallel child workflows, external Agent calls, Runtime Skill execution, and plugin/MCP calls require current host evidence and user authorization.
- Plan Mode UI support is unavailable in current evidence.
- Goal Mode evidence is limited.
- OpenCode and Gemini are unavailable in current evidence.
- PalBench and release records are qualitative evidence, not statistical benchmark proof.
