## Pal Asset Execution Contract

v0.5 now includes the no-code Pal Asset Execution Contract:

- `core/pal-asset-execution-contract.md`
- `core/asset-loading-gate.md`
- `templates/pal/task-asset-packet.md`
- `templates/pal/asset-use-summary.md`
- `templates/pal/missing-asset-plan.md`
- `docs/07-pal-asset-execution/README.md`

This contract explains that a Pal is not only a name, persona, prompt wrapper,
or direct tool call. For substantive work, a Pal should identify task-relevant
assets, load the relevant assets, form a Task Asset Packet, use host tools only
after Pal judgement has shaped the request, verify with a quality gate, and
report an Asset Use Summary.

R199 recorded real Codex host read-only rehearsal for:

- asset-grounded user custom Pal product review;
- no blind ImageGen/tool call design handling;
- PalSmith completeness judgement;
- lightweight typo-fix contrast;
- Quinn review.

R200 recorded controlled write rehearsal on a test fixture. The controlled
write stayed inside the fixture and evidence paths. The real user custom Pal,
official Pals, and central contacts stayed unchanged.
