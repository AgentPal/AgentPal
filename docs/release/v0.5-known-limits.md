# AgentPal v0.5 Known Limits

Status: R211 release-prep known limits.

## No Runtime Backend

AgentPal v0.5 is a no-code Pal layer and organization workspace. It does not
add AgentPal's own runtime backend, daemon, scanner, connector backend, CLI,
installer, validator service, or app runtime.

Execution still belongs to Codex, Claude Code, OpenCode, OpenHands, or another
host runtime.

## No Marketplace Backend

Marketplace metadata and listing requests remain planning / draft materials.
AgentPal v0.5 does not include a live Marketplace backend or public listing
surface.

## No Automatic Contacts Write

Central contacts are still the official bundled Pal roster source. User custom
Pals do not enter central contacts automatically. Contacts registration requires
a separate governance task and explicit authorization.

## User Custom Pal Discovery Is Explicit

User custom Pals are private by default. Discovery, invocation, limited
delegation, contacts registration, Marketplace draft metadata, and public
listing requests are separate permission fields.

## Scoped Certification Only

R209/R210 provide one scoped-certified-with-notes representative task-family
record for each official bundled Pal.

This is not full Pal certification. It is not all-task-family certification.
It does not certify every runtime, tool path, environment, missing-asset path,
or publication path.

## Evidence Limits

Some evidence is real host read-only evidence, dry-run evidence, or controlled
fixture evidence. Those labels must stay visible and must not be upgraded into
live-write, remote-publish, or public-release evidence.

## Non-Codex Host Limits

Codex-first evidence is stronger than non-Codex host evidence. Claude Code,
Generic CLI Agent, OpenCode, OpenHands, or other hosts may need their own
fresh-host checks before stronger public compatibility claims.

## No Live Online Update Backend

AgentPal v0.5 does not add a live online update backend. Any future update,
sync, marketplace, or distribution service requires a separate design,
implementation, security review, and user authorization.

## Remote Publication Not Included

R211 does not perform `git push`, `git pull`, `git fetch`, tag creation, tag
push, GitHub Release creation, or release asset upload.
