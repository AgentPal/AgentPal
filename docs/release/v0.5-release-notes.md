# AgentPal v0.5 Release Notes

Status: final prepared local release notes.

These notes are prepared locally for a future user-authorized release. No remote
release has been executed by R212.

## Summary

AgentPal v0.5 is a no-code Pal layer for organizing AI teams on top of existing
Agent runtimes such as Codex, Claude Code, OpenCode, and OpenHands.

v0.5 focuses on PalSmith, custom Pal governance, Pal Asset Execution, and scoped
release-readiness evidence.

## PalSmith v0.5

PalSmith v0.5 supports no-code Pal creation and governance. It can help prepare:

- composite Pal creation plans;
- draft Pal Packs;
- user custom Pal installation plans;
- authorization and revocation records;
- Pal Team designs;
- quality and release-readiness reviews.

PalSmith can model:

- human or source-inspired thinking style;
- knowledge and source boundaries;
- role and job responsibility;
- personality, tone, and expression style;
- Skill / Agent / workflow / memory composition;
- contacts and Marketplace metadata planning.

This is a planning and asset-governance capability. PalSmith does not
automatically generate, install, register, publish, or execute a Pal without
host-runtime evidence and user confirmation.

## Draft Pal Pack To User Custom Pal Flow

v0.5 documents the path from reviewed draft Pal Pack to user custom Pal:

1. Create or review a draft Pal Pack.
2. Keep the draft outside central contacts and outside `official/pals/`.
3. Use the draft-to-custom installation flow only after review.
4. Keep the installed user custom Pal private by default.
5. Enable discovery, delegation, contacts registration, or public listing only
   through separate explicit authorization.

## User Custom Pal Authorization Model

User custom Pals are default-private. The following permissions remain separate:

- discovery;
- invocation;
- limited delegation;
- contacts registration;
- Marketplace draft metadata;
- public listing request.

Opening one field does not open another. Revocation and expiry evidence is
recorded through R191; it remains read-only / dry-run evidence unless a future
task explicitly authorizes live write execution.

## Pal Asset Execution

v0.5 includes the Pal Asset Execution Contract:

- `core/pal-asset-execution-contract.md`
- `core/asset-loading-gate.md`
- `templates/pal/task-asset-packet.md`
- `templates/pal/asset-use-summary.md`
- `templates/pal/missing-asset-plan.md`
- `templates/pal/scoped-certification-record.md`

The contract says a Pal should work from task-relevant identity, knowledge,
Skill, workflow, runtime policy, memory, and quality assets rather than only
from a name, persona, prompt wrapper, or direct tool call.

## Scoped Certification Status

R209 and R210 added scoped certification records for representative official
Pal task-family scopes.

All bundled official Pals have at least one scoped-certified representative
task-family record.

This is not full Pal certification and not all-task-family certification.

## Known Limits

- No AgentPal runtime backend.
- No CLI or installer.
- No scanner, daemon, connector backend, validator service, or app runtime.
- No Marketplace backend or live public listing.
- No automatic central contacts write.
- User custom Pal discovery requires explicit authorization.
- Scoped certification is per Pal + task family.
- Non-Codex host support remains evidence-limited.
- Some evidence is real host read-only, dry-run, or controlled fixture evidence.

See `docs/release/v0.5-known-limits.md`.
