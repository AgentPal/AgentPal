# AgentPal v0.5 Release Preflight Checklist

Status: R212 local release package preflight.

This checklist is for local release package preparation. It does not authorize
remote operations.

## Local Repo Status

- [ ] `git status --short --branch` reviewed.
- [ ] Working tree contains only expected R212 release package docs, evidence,
  and index updates before commit.
- [ ] Post-commit working tree is clean.

## JSON Checks

- [ ] `agentpal.json` parses as JSON.
- [ ] `official/pals/PalSmith-pal-governor/pal.json` parses as JSON.
- [ ] `workspace/resources/user-pals/FirstPrinciplesProductReviewer/pal.json` parses as JSON when present.
- [ ] `evals/palbench/v0.5/asset-usage/controlled-write-fixtures/FirstPrinciplesProductReviewerControlledWrite/pal.json` parses as JSON when present.

## Markdown Link Checks

- [ ] New release package docs have 0 missing local links.
- [ ] R212 evidence docs have 0 missing local links.
- [ ] Modified release docs have 0 missing local links.

## Roster And Boundary Checks

- [ ] Official Pal directory count remains 10.
- [ ] `workspace/organization/contacts/` has no unauthorized diff.
- [ ] `official/pals/` has no unintended diff.
- [ ] Real user custom Pal files have no unintended diff.
- [ ] Controlled-write fixture is still a test artifact.
- [ ] Luma is not modified or upgraded.

## No-Code Boundary Scan

- [ ] No runtime backend introduced.
- [ ] No CLI or installer introduced.
- [ ] No scanner / daemon / connector backend introduced.
- [ ] No Marketplace backend introduced.
- [ ] No automatic contacts write introduced.
- [ ] User custom Pal discovery remains explicit-authorization-only.

## Overclaim Scan

Check that the release package does not claim:

- all task families verified;
- all official Pals fully certified;
- full certification completed;
- Marketplace is live;
- runtime backend is implemented;
- GitHub Release completed;
- tag pushed;
- release asset uploaded.

Forbidden phrases may appear only in boundary, known-limits, or negative-test
contexts.

## Release Notes Final Review

- [ ] `docs/release/v0.5-release-notes.md` exists.
- [ ] It includes PalSmith composite Pal creation.
- [ ] It includes draft-to-custom flow.
- [ ] It includes user custom Pal authorization / revocation boundary.
- [ ] It includes Pal Asset Execution adoption.
- [ ] It includes scoped certification status with limits.
- [ ] It states no remote release has been executed.

## Known Limits Final Review

- [ ] `docs/release/v0.5-known-limits.md` exists.
- [ ] It covers no runtime daemon / backend.
- [ ] It covers no Marketplace backend.
- [ ] It covers no automatic contacts write.
- [ ] It covers no all-task-family certification.
- [ ] It covers no full Pal certification.
- [ ] It covers no live online update backend.
- [ ] It covers explicit user authorization for custom Pal discovery.
- [ ] It covers explicit user authorization for push / tag / release.

## Package Manifest Check

- [ ] `docs/release/v0.5-release-package-manifest.md` exists.
- [ ] It names target version `v0.5`.
- [ ] It states readiness verdict `ready_with_notes`.
- [ ] It lists included docs / prompts / templates / evidence categories.
- [ ] It states official Pal count = 10.
- [ ] It states remote operations require user authorization.

## User Authorization Gate Before Remote Operations

Do not perform any of these unless the user explicitly authorizes them in a
later task:

- `git push`
- tag creation
- tag push
- GitHub Release creation
- release asset upload
- package registry publication
