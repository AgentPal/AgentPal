# AgentPal v0.5 Local Development Scope

Date: 2026-06-28

## Scope

AgentPal v0.5 local development starts from the completed v0.4 central organization workspace foundation. The v0.5 theme is Org Pack / AI Organization Package foundations for reusable no-code organization assets.

This is a local development scope freeze. It is not GitHub sync, not a tag, and not a GitHub Release.

## v0.5 Focus

v0.5 focuses on:

- Org Pack / AI Organization Package standards
- reusable organization packages
- customer-private versus reusable asset boundaries
- FDE / expert delivery pack foundations
- cross-organization collaboration prerequisites
- no-code governance, verification, and memory policies

v0.5 continues the v0.4 boundaries:

- external projects stay thin-bound
- AgentPal central assets stay in the AgentPal Workspace
- Pal roster truth stays in central contacts
- ownership and routing remain AI judgement, not keyword routing
- host runtimes execute real actions and must provide evidence

## Out Of Scope

v0.5 local development does not add:

- GitHub sync
- GitHub Release
- tag
- marketplace / hub program
- CLI
- Web App
- Desktop App
- installer
- scanner
- validator
- daemon
- database
- auto sync
- auto execution engine
- connector or external business system API client
- automatic Runtime / business system discovery
- keyword router or deterministic semantic router

## Org Pack Boundary

An Org Pack is a no-code organization asset package. It may contain reusable organization descriptions, recommended Pals, workflow candidates, governance rules, capability references, memory policy, verification checklists, and public-safe examples.

An Org Pack is not a Pal Pack, not a project template, not an installer, not a connector, not a database, not a marketplace program, not a scanner, not a validator, and not a keyword router.

## External Project Boundary

External projects still receive only thin binding entry points by default:

- `.agentpal/project.json`
- `.agentpal/AGENTPAL.md`
- protected root instruction blocks
- optional Claude Code local settings

Org Pack assets must not be copied into external project `.agentpal/org-pack/`, `.agentpal/pals/`, `.agentpal/memory/`, `.agentpal/reports/`, `.agentpal/workflows/`, `.agentpal/evals/`, `.agentpal/capability-inventory/`, or `.agentpal/business-systems/` by default.

## Central Roster Boundary

Org Packs may recommend Pals, but recommendations are AI judgement inputs only. They are not routes, not assignments, and not central roster edits.

The central roster remains:

- `workspace/organization/contacts/pals.json`
- `workspace/organization/contacts/PAL_CONTACTS.md`
- `official/pals/`

## Customer-Private Boundary

Reusable Org Packs must not contain customer private data, customer credentials, real project secrets, customer system identifiers, private project memory, or private evidence.

Customer-specific facts belong in private central project records, customer-private records, or explicitly approved private evidence locations, not in a reusable public Org Pack.
