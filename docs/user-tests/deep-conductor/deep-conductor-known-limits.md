# DeepConductor Known Limits

## Current Limits

- DeepConductor is a no-code orchestration and verification protocol, not an automatic multi-agent runtime engine.
- AgentPal does not add a backend, daemon, scanner, validator, connector, CLI product, or Marketplace backend.
- A test can pass as `pass_with_notes` when the host cannot prove a strict project-bound thread.
- R241 recorded this exact note: the fresh project filesystem walkthrough passed, but strict Codex UI project-bound thread loading was not proven.

## Not-Run By Default

Unless the user explicitly authorizes them, these are not run:

- live GitHub outreach;
- public posting;
- GitHub Release;
- push / pull / fetch / tag;
- live external user validation;
- live source research;
- user contact or recruitment execution.

## Claims To Avoid

Do not claim:

- full certification completed;
- all Pal task families verified;
- Marketplace is live;
- runtime/backend is implemented;
- DeepConductor automatically runs external agents;
- live external validation is complete;
- a GitHub Release was created.

## What A Passing Manual Test Means

A passing manual test means a user can see and judge the DeepConductor chain. It does not mean AgentPal has become a runtime engine or a released marketplace product.
