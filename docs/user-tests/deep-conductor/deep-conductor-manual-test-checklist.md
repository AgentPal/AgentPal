# DeepConductor Manual Test Checklist

Use this checklist while reading the output.

## Setup

- [ ] AgentPal thin binding is enabled in the current project.
- [ ] The active project is the external test project.
- [ ] The user did not need to name internal files, protocols, or R numbers.

## Main Request

- [ ] A natural-language request triggers DeepConductor.
- [ ] Task Intake appears.
- [ ] Team First Discovery appears before any new-team creation.
- [ ] An existing Team Pack is considered.
- [ ] A fitting Team Pack is reused when available.
- [ ] The answer explains why selected Team / Pal participants fit.
- [ ] The answer explains why rejected Team / Pal candidates are not used.
- [ ] Each selected Pal has a Work Plan.
- [ ] Each selected Pal has Asset Preflight.
- [ ] The Execution Trace maps work back to owners.
- [ ] Parallel / sequential / sub-agent decision has a reason.
- [ ] Owner Assignment Integrity Gate runs.
- [ ] Closure Gate runs.
- [ ] Quinn final verification runs.
- [ ] The final deliverable is directly usable or clearly marked not-ready.

## Boundary Pressure Request

- [ ] User-specified wrong assignments are shown.
- [ ] Faye as routine promotion executor is rejected.
- [ ] Quinn as ordinary copywriter is rejected.
- [ ] PalSmith as routine test-plan executor is rejected when an existing team fits.
- [ ] The corrected assignment is shown.
- [ ] The answer says whether execution continues.

## Boundary Claims

- [ ] The answer does not claim full certification.
- [ ] The answer does not claim Marketplace is live.
- [ ] The answer does not claim runtime/backend/daemon/scanner is implemented.
- [ ] The answer does not claim GitHub Release, push, tag, or live external validation happened.
- [ ] Not-run actions are labelled as not-run, skipped, or requiring user authorization.
