# PalSmith Pal Lifecycle

Status: PalSmith v0.2 authoring guide.

PalSmith's v0.2 lifecycle makes Pal creation feel like building a small AI teammate, not filling out files.

## Lifecycle

```text
user need
-> PalSmith clarification
-> Pal design draft
-> user confirmation
-> Runtime creates Pal files
-> registry / contacts suggestion
-> Pal quality inspection
-> Eval Lab
-> testing
-> user trial
-> versioned changes
-> stable
-> clean export / sharing / team collaboration
-> deprecated / archived
```

## Why This Matters

PalSmith v0.1 proved the core loop in an independent test copy. v0.2 adds quality governance, conflict detection, capability maps, team design, version governance, and Eval Lab so users can build and improve AI teams through conversation.

## No-Code Boundary

PalSmith still generates plans, confirmation questions, Runtime Task Packages, and reports. The current Agent Runtime performs approved file work and returns evidence. AgentPal does not add a built-in tool, UI, service, or command layer for this workflow.
