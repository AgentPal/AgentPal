# Nova Product / Strategy Gap Report

## PalSmith-style Inspection Scope

Target: `pals/Nova-product`

Inspection date: 2026-06-26

Method: PalSmith job-fitness inspection using Nova current assets, PalSmith quality protocols, current contacts/registry, default Pal boundary assets, Harper collaboration boundary, and external product-management source research.

## Current Positioning Problem

Nova was present as a product manager and idea-structuring Pal with useful foundations for idea intake, problem definition, scenarios, scope, MVP slicing, PRD writing, prioritization, acceptance criteria, developer handoff, risk review, decision logs, and feedback synthesis.

The earlier asset set did not yet express Nova as AgentPal's official Product / Strategy Lead Pal. It was too close to "product advice and PRD helper" and lacked source-backed depth for product discovery, Jobs-to-be-Done, value proposition, roadmap reasoning, metrics, feedback loops, go-to-market alignment, and explicit cross-Pal boundaries after Harper, Morgan, Atlas, Vega, Quinn, Rhea, Mira, and PalSmith were deepened.

## Existing Strengths

- Existing directory and direct call: `pals/Nova-product`, `/pal Nova`.
- Existing 12 foundational skills and several product knowledge cards.
- Existing product output contract and early no-code/execution boundary.
- Existing collaboration language that rejects fixed routing.

## Research Questions

- What makes a useful modern PRD and product brief?
- How should product discovery keep teams focused on evidence and user outcomes?
- How should user segments, JTBD, and value propositions shape product scope?
- How should prioritization avoid feature-list thinking?
- How should roadmaps stay outcome-based rather than output-only?
- Which product risks must be checked before implementation or release?
- How should metrics and feedback loops connect to product goals?
- What collaboration boundary keeps Nova from absorbing research, writing, document, quality, safety, implementation, and Pal governance work?
