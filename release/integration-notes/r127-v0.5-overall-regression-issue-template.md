# R127 - v0.5 Overall Regression Issue Template

R128 should record every finding in this shape. Use `missing` or `not-run` instead of substituting evidence.

| issue_id | severity blocker/high/medium/low/note | test_group | affected_path | finding | expected | actual | recommended_fix | safe_to_fix_in_R128 yes/no | requires_R129_fix_round yes/no | status |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| R128-ISSUE-001 | blocker | Example group | `path/or/missing` | Finding summary | Expected condition | Actual evidence | Recommended action | no | yes | open |

## Severity Guide

| Severity | Use When | Default Action |
| --- | --- | --- |
| blocker | JSON invalid, core path missing, clean-copy failure, customer-private/credential leak, external project unsafe write, active fixed keyword routing, scanner/connector/marketplace/runtime program expansion, tag/push/release attempt, or false completion claim | Stop completion claim and open R129 fix round. |
| high | Official Pal, PalSmith pilot, Pal Asset, Org Pack, FDE Pack, Combined Pack, or central-roster contract drift that can mislead users | Prefer R129 fix round unless the fix is tiny and isolated. |
| medium | Documentation, navigation, example, or validation mismatch without safety impact | May fix in R128 if scoped and recorded. |
| low | Typo, formatting, or non-blocking wording issue | May fix in R128. |
| note | Observation that does not require change | Record only. |

## R128 Fix Rules

Set `safe_to_fix_in_R128` to `yes` only when:

- severity is medium or low;
- the fix is a narrow text, path, or JSON pointer correction;
- the fix does not change central roster, official Pal metadata, standards semantics, runtime claims, release status, or user/private data boundaries;
- the R128 result file records the exact fix.

Set `requires_R129_fix_round` to `yes` when:

- severity is blocker or high;
- the fix requires changing standards, official Pal metadata, central roster, public package boundaries, or release posture;
- the finding needs stakeholder review or broader evidence.

## Status Values

Use one of:

- `open`
- `fixed_in_R128`
- `deferred_to_R129`
- `false_positive_reviewed`
- `accepted_note`
