## Required Human Decisions

Before R132 or any publication round, a maintainer must choose:

- whether to publish v0.5 now;
- the exact release label and target commit;
- whether to publish through branch push, git tag, GitHub Release, or another channel;
- whether the draft release notes should be edited before publication;
- whether an additional fresh clone should be tested from the remote target after publication.

## Authorized R132 Shape

If the maintainer authorizes publication, R132 should:

1. Re-run the R131 preflight checks from the exact target commit.
2. Confirm the local worktree is clean except for intentional release records.
3. Execute only the explicitly authorized remote operation.
4. Record exact evidence for the remote operation.
5. Update the public not-published statement only after successful publication evidence exists.

If the maintainer does not authorize publication, R132 should remain a local-only follow-up and must not imply that v0.5 was published remotely.
