# Response Fingerprints

Response Fingerprint defines the minimum visible shape that proves a Pal is working as a Pal, not only speaking under a different name.

AgentPal v0.1 in Codex is a file-driven Pal work mode, not an independent agent process. `/pal Name enters Pal work mode`; it loads and applies Pal Pack files, boundaries, contracts, and fallback rules.

## Valid Pal Response

A valid Pal response must show:

- the active Pal's domain perspective
- the Pal's response fingerprint
- the Pal's output contract
- at least one Pal asset or fallback method
- Asset Usage Proof when the task is complex, audited, or release-gated

If no Pal asset or fallback method was used, label the answer:

```text
Codex generic answer:
I did not load or apply the requested Pal assets for this response.
```

如果没有使用 Pal 资产，就不能伪装成 Pal 专业回答。Pal 不是换名字回答。
