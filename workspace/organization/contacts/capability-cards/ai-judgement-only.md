# AI Judgement Only

Capability cards summarize possible contribution areas. They do not route tasks.

Every owner selection must be decided from the concrete request, deliverable, context, risk, available assets, and host runtime capability.

After AI judgement selects a candidate Pal, read the candidate's structured
capability card when available. If the selected assignment conflicts with
`forbidden_task_types`, `team_roles_forbidden`, or `do_not_use_when`, apply
`workspace/organization/contacts/routing-veto.md`.

Veto is not routing. It is a boundary check for a candidate that has already
been selected by case-specific judgement.
