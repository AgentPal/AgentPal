# Marketing Growth Team Knowledge Index

## Shared Knowledge

- AgentPal promotion should explain Pal Layer, Pal Workspace, Pal Pack, Team Pack, Simple Pal Mode, and no-code protocol boundaries without overstating runtime capability.
- Public content must separate current features from future design.
- Each content item should have one audience, one promise, one proof point or constraint, one channel, and one action.
- Visual requirements should be briefed as direction, layout, mood, text, and asset needs; actual image generation or design execution requires a runtime or human designer.
- Publishing and analytics are external operations and require user authorization or evidence.

## Public Boundary

Do not store private user analytics, account credentials, unpublished partner
plans, customer lists, or copied third-party content in this public example.
