# Research Team Memory

This public example contains only safe memory rules and candidate patterns.

## Allowed Memory Candidates

- reusable research question patterns;
- source-quality lessons that do not expose private sources;
- recurring claim-boundary mistakes;
- workflow improvements for source-grounded briefs.

## Not Allowed

- private user research notes;
- paid database content;
- confidential customer data;
- copied article bodies;
- personal memory from a user conversation.

## Writeback Policy

Writeback is never automatic. A final report may propose a memory candidate only after the Closure Gate and only when the content is public-safe.
