# Sales CRM Delivery Pack Example

This is a public-safe near-runnable Delivery Pack example for a fictional B2B sales team.

It demonstrates how Faye, the AI Delivery Pal, can turn a sales operations need into an auditable no-code Delivery Pack for lead organization, CRM follow-up planning, customer segmentation, sales copy drafting, reminder cadence, and deal review.

## Public-safe Boundary

This pack contains no real customer names, real CRM exports, real phone numbers, real emails, real deal amounts, credentials, tokens, cookies, webhooks, or external CRM API calls.

All data examples are fictional placeholders. Real CRM data belongs in customer-private project records, not in this reusable pack.

## Pack Shape

- `DELIVERY.md`: delivery overview and operating model
- `PROJECT.md`: fake business context, assumptions, data needs, and risks
- `PAL_TEAM.md`: candidate Pal team blueprint
- `FAYE_BUILD_REQUEST.md`: request from Faye to PalSmith
- `FLOWS/`: five sales CRM flow packs
- `TASKS/`: first reusable task package templates
- `INTEGRATIONS/`: manual handoff notes only
- `ACCEPTANCE.md`: acceptance criteria
- `TRAINING.md`: user training outline
- `REPORTS/`: report placeholders
- `first-task-package.example.md`: first task package example

## Candidate Use

Use this example when testing whether a Delivery Pack can represent a sales CRM workflow without becoming a connector, CRM client, scanner, marketplace package, or automatic execution engine.

Candidate Pals in this pack are AI judgement inputs only. They are not route maps or fixed assignments.
