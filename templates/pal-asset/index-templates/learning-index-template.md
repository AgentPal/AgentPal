# Learning Index

## Purpose

This directory lists candidate assets and lessons under review for `[Pal name]`.

Learning is a candidate asset queue. Items require promotion review before becoming knowledge, Skill, workflow, runbook, memory, eval, or example assets.

## What Belongs Here

- Candidate Skills.
- Candidate workflows.
- Candidate runbooks.
- Candidate knowledge notes.
- Candidate memory notes.
- Feedback and observations that need review.
- Rejected or deferred candidates with reasons when useful.

## What Does Not Belong Here

- Final stable knowledge.
- Approved Pal Skills.
- Full reports with no extracted lesson.
- Raw private material.
- Credentials, tokens, secrets, or private evidence.
- Runtime state.
- Keyword route maps or deterministic dispatch tables.

## Review Notes

- Every candidate should eventually be promoted, revised, rejected, archived, or left as `needs-review` with a reason.
- Promotion requires human or owner-Pal review when behavior, public safety, or skill boundary changes.
- Do not infer that a learning candidate is already approved capability.

## Public Safety Boundary

Do not place private user content, customer identifiers, secrets, credentials, tokens, or private project evidence in a public learning index.

## External Project Boundary

Learning assets belong in the central Pal Pack by default. Do not write them into external project `.agentpal/` by default.
