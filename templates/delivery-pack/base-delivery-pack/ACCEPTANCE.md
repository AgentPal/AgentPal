# Acceptance

## Acceptance Criteria

| Criterion | Status | Evidence |
| --- | --- | --- |
| business goal is defined | `missing` | `PROJECT.md` |
| Flow Pack list is defined | `missing` | `FLOWS/` |
| first Task Package is defined | `missing` | `TASKS/` |
| Pal Team Blueprint is reviewed | `missing` | `PAL_TEAM.md` |
| Faye Build Request is reviewed | `missing` | `FAYE_BUILD_REQUEST.md` |
| human review owner is named | `missing` | private project record if needed |
| customer-private boundary is preserved | `not-run` | public-safety review required |

## Status Terms

Use `pass`, `partial`, `missing`, `not-run`, `blocked`, and `requires-human-review`.

Do not turn `missing`, `not-run`, or `requires-human-review` into `pass`.
