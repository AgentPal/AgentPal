# Customer-Private Assets Boundary

Reusable Org Packs must not include customer-private assets.

Customer-private assets include customer data, real project documents, private evidence, credentials, private system identifiers, customer-specific business rules, and private project memory.

Public reusable Org Packs may include de-identified examples, public-safe patterns, workflow candidates, governance rules, and checklists.

When in doubt, keep the asset private and reference it from an approved private project record instead of placing it in this reusable template.
