# Lifecycle Workflow

This is the Pal's lifecycle workflow template.

## Steps

1. Intake: confirm user goal, context, and output requirements.
2. Boundary: confirm responsibility, risks, and non-goals.
3. Asset: select a Skill, Knowledge Card, Runbook, Workflow, or fallback method.
4. Draft: produce domain judgement, output, or Task Package.
5. Evidence: request or review evidence.
6. Learning: record knowledge gaps or repeated task candidates.
