No-code AI organization layer

给现有 Agent，装上一支会协作的 Pal 团队。

AgentPal 是一个 Markdown / JSON / protocol 工作区。它不替代 Codex、Claude Code 或 OpenCode，而是给这些 Runtime 加上一层可维护的团队组织：角色、知识、Skill、流程、上下文、记忆边界和验收习惯。

不是更多按钮，是更清楚的分工。

AgentPal 把“能力”“角色”和“执行层”拆开。Skill 让 AI 会做某类事，Pal 让 AI 知道自己是谁、负责什么、该如何协作，Agent Runtime 负责真实读写和工具调用。

01 / SKILL

能力包

可复用的提示词、流程、工具说明或专业方法。它让 AI 更会做某类任务，但通常不承担长期职责、记忆和团队分工。

02 / PAL

工作伙伴

带身份、职责、知识、Skill、流程、记忆边界和输出契约的团队成员。Pal 会判断自己该做什么，也知道什么时候该交给别人。

03 / AGENT

执行运行层

Codex、Claude Code、OpenCode、OpenHands 等 Runtime 执行真实文件操作、命令、工具调用和外部系统动作。AgentPal 不假装替它们执行。

从一句目标，到一个可交付工作组。

AgentPal 的重点不是让所有角色都变成独立运行的 Agent，而是先把团队角色、上下文、任务边界和验收方式组织清楚。复杂任务再按需要交给 Runtime 或外部工具执行。

MIRA FIRST

你只需要先说目标。

Mira 会判断这是普通回答、专业 Pal 任务、复合交付，还是需要 Task Package。你也可以直接用 /pal Name 调用专业 Pal。

1. 判断任务形状

明确目标、交付物、缺失信息、风险、上下文和候选 owner Pal。

2. 装载相关资产

只读取当前任务需要的 Pal 身份、知识、Skill、Workflow、输出契约或 fallback 方法。

3. 形成执行边界

把任务包、验收标准、禁止事项和证据要求交给当前 Runtime，而不是让 Pal 冒充执行层。

4. 回收结果与学习

由 owner Pal 汇总执行证据、风险和下一步；必要时把可复用经验沉淀为 Pal 资产。

轻量，也要诚实。

AgentPal 的价值来自组织层，而不是伪装成全能后台。它让现有 Runtime 更会协作，但不会隐式扫描机器、安装工具、连接外部系统或自动发布。

AgentPal 提供什么

  • Pal 身份、职责、知识、Skill 和 Workflow。
  • Mira-first 协调、直接 Pal 调用和复合任务判断。
  • Context slicing、Task Package、验证习惯和边界声明。
  • PalSmith / Faye 支持自定义 Pal 团队和交付包设计。
  • Thin binding，把真实项目轻量接入 AgentPal Workspace。

AgentPal 不声称什么

  • 不是独立 App、CLI、daemon、scanner 或 connector backend。
  • 不自动修改 contacts，不自动发布 Marketplace，不自动推送 Git。
  • 不替代 Codex、Claude Code、OpenCode 等执行层。
  • 不在没有证据时声称工具、插件、MCP 或外部 Agent 可用。
  • 不把私有项目记忆写进公开 Pal Pack 或示例资料。
