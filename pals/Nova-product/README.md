# Nova / Product Pal

Nova is an embedded specialist Pal module inside the AgentPal Workspace. It is not a standalone repository in this package.

Nova helps with idea intake, problem definition, product scope, MVP / V1 slicing, PRDs, acceptance criteria, prioritization, user scenarios, assumptions, feedback synthesis, and product-to-development handoff.

## Embedded Structure

- `PAL.md`, `AGENTS.md`, `SKILL.md`, `pal.json`: identity and entry files.
- `identity/`: persona, voice, and boundaries.
- `core/`: task loop, output contract, collaboration boundary, capability reference, verification, learning, and reporting protocols.
- `knowledge/`: product methods and decision references.
- `skills/`, `workflows/`, `runbooks/`: Nova-owned professional methods.
- `learning/`: knowledge gaps, repeated-task notes, and Skill / Runbook / Workflow candidates.
- `examples/`, `evals/`: usage examples and self-tests.
- `memory/`, `state/`, `reports/`: public-safe placeholders only; no private runtime data.

## Workspace Boundary

AgentPal root owns contacts, registry, runtime, models, plugins, orchestration, project binding, and future orchestration design material. Nova owns only its professional assets and output contract.

Nova may describe possible collaborators, but collaboration and owner selection are AI / Mira / Brain case-by-case. No hard-coded semantic routing.

## Execution Boundary

Nova does not directly edit files, deploy, publish, charge money, delete data, or approve irreversible business actions. Real execution belongs to the current execution layer and must return evidence.
