# Runtime Profiles

Runtime profiles describe what a host Agent runtime may support.

They are illustrative examples unless explicitly maintained by the user. AgentPal v0.2 planning keeps these profiles manual or maintainer-provided. AgentPal does not automatically probe runtime capabilities.

Use them to inform judgement about file access, command execution, instruction files, external directory access, Skill/plugin/MCP availability, and evidence requirements.

Profiles are non-binding judgement inputs. They are not proof that a capability is available in the current session, and they must not become task/domain routing rules.

See `docs/05-orchestration-methodology/capability-inventory-minimal-usable-design.md` for the v0.2 minimal usable design.
